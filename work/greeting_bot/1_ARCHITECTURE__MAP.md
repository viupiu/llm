## 0. Профиль целевой аудитории

**Типичный пользователь:**
- Возраст: 18–55 лет
- Стиль речи: бытовой, непринуждённый, может использовать сленг и сокращения
- Техническая грамотность: средняя — не привык к формальным интерфейсам
- Контекст использования: пишет боту с чётким запросом "придумай поздравление для [кого-то]", ожидает готовый вариант
- Характер запросов: от односложных ("для мамы", "для друга") до развёрнутых ("для тёщи на юбилей")

---

## 1. Навыки (Skills)

### Skill: `EVENTS`
Инфраструктурный навык. Содержит системные точки входа: Загрузка.

### Skill: `GREETING_MAIN`
Бизнес-навык — основной сценарий поздравлений. Содержит 17 узлов: вопрос-корень, 13 обработчиков поздравлений, благодарность, прощание, fallback.

### Skill: `*` (Fallback)
Глобальный fallback навык. Содержит универсальный узел для всего, что не попало ни в один бизнес-узел.

## 4. Логика сценария (Переходы)

### Основной поток

```
[Загрузка бота]
    └→ E1: Start_Load (EVENT)
       "Здравствуйте! Для кого нужно придумать поздравление?"
       [%that_anchor="Для кого нужно придумать поздравление?"]
    └→ Пользователь отвечает
       ├→ G2: Для мужчин
       ├→ G3: Для мужчин близких
       ├→ G4: Для мужчин малознакомых
       ├→ G5: Для женщин
       ├→ G6: Для женщин близких
       ├→ G7: Для женщин малознакомых
       ├→ G8: Для детей
       ├→ G9: Для старших мужчин
       ├→ G10: Для старших женщин
       ├→ G11: Близкие (универсальное)
       ├→ G12: Мало знакомые (универсальное)
       ├→ G13: Универсальное (без деталей)
       ├→ G14: Благодарность
       ├→ G15: Прощание
       └→ G16: Fallback *
          "Не понял, для кого вам поздравление? Подскажите подробнее."
          [%that_anchor="Для кого нужно придумать поздравление?"]
          → (возврат к вопросу)
```

### Поведение после поздравления

После выдачи позравления бот ожидает нового сообщения пользователя:
- **Благодарность (G14):** "Рад помочь!"
- **Прощание (G15):** "До свидания!"
- **Новый запрос:** Бот выдаёт поздравление снова (цикличный режим)
- **Fallback (G16):** Возврат ко вопросу

### Поведение при Fallback (G16)

Узел G16 сработал, но пользователь продолжает отвечать — бот повторяет вопрос и ждёт уточнения. Счётчик не используется (по ТЗ не требуется).

## 5. Грани и риски пересечений

### Kритические грани

| Узел A | Узел B | Риск | Разделение |
|--------|--------|------|-----------|
| G2 (мужчины) | G3 (мужчины близкие) | Высокий | G3 содержит маркеры близости ("любимый", "дорогой"), G2 — нейтральные |
| G2 (мужчины) | G4 (мужчины малознакомые) | Высокий | G4 — формальное обращение ("начальник", "коллега") |
| G3 (мужчины близкие) | G11 (близкие универсальное) | Средний | G3 содержит маркер мужского пола + близости, G11 — только близость |
| G8 (дети) | G2/G5 (мужчины/женщины) | Средний | G8 — маркеры детского возраста, G2/G5 — взрослые |
| G9/G10 (старшие) | G2/G5 (мужчины/женщины) | Средний | G9/G10 — маркеры старшего возраста + пола |
| G13 (универсальное) | все остальные | Высокий | G13 — default, когда пол, возраст, близость не указаны |
