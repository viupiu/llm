# Архитектурная карта: Генератор оправданий

**BotSlug:** `excuse_gen_bot`
**Канал:** чат (текстовый)
**Дата:** 2026-05-29

---

## 0. Профиль целевой аудитории

Типичный пользователь — молодой взрослый (18–35 лет), активно использующий мессенджеры и соцсети. Стиль речи неформальный, разговорный, с допустимым использованием сленга, сокращений и эмоциональных конструкций. Уровень технической грамотности выше среднего, но бот не требует никаких специальных знаний. Пользователь ожидает быстрый, смешной, «мемный» результат без излишней формальности. Запросы часто однострочные, императивные или в форме вопроса («опоздал на совещание», «почему не написал», «не сделал задачу — придумай»).

---

## 1. Навыки (Skills)

### Skill: EVENTS

Инфраструктурный навык. Содержит EVENT-узел Загрузки — первую точку входа любого бота.

### Skill: EXCUSE_GEN

Основной бизнес-навык. Содержит узел-задаватель, узлы-обработчики интентов, узел-задаватель повторного запроса «Ещё отмазку?», дерево ответов да/нет/фоллбэк для повторного запроса, и дерево-фоллбэк для основного вопроса. Вся логика генерации оправданий и циклического взаимодействия.

### Skill: FALLBACK_GLOBAL

Глобальный fallback-навык. Содержит один `*`-узел, который ловит всё, что не попало ни в один обработчик. Переводит пользователя в основной сценарий.

## 4. Логика сценария

### Основной поток

1. **EVENT Загрузки** (`Загрузка`, узел 1) — первая точка входа любого бота. Системный триггер `EVENT 00b2fcbe-f27f-437b-a0d5-91072d840ed3`. Бот произносит краткое приветствие и переводит управление: `[@goto("Какая у тебя беда?")]`.

2. **Узел-задаватель (`Какая у тебя беда?`, узел 2) произносит вопрос («Какая у тебя беда?») и устанавливает `%that_anchor="Какая у тебя беда?"`. Ждёт ответа пользователя.

3. **Пользователь отвечает** — его ответ обрабатывается одним из обработчиков (узлы 3–6):
   - Бот генерирует смешное оправдание, соответствующее интенту.
   - В ответе используется механизм anti-repeat через `:lu:` (limit usage) или `[udict(name)]` — автоматическое перебирание элементов словаря без повторов в рамках цикла/сессии.
   - После генерации переход: `[@goto("Ещё отмазку?")]` — бот спрашивает, хочет ли пользователь ещё одну отмазку.

4. **Узел-задаватель (`Ещё отмазку?`, узел 9) произносит вопрос («Хочешь ещё одну отмадку?») и устанавливает `%that_anchor="Ещё отмазку?"`. Ждёт ответа пользователя.**

5. **Обработка ответа на «Ещё отмазку?»:**
   - **Да (узел 10):** Пользователь соглашается. Переход к основному вопросу: `[@goto("Какая у тебя беда?")]`. Цикл повторяется с шага 2.
   - **Нет (узел 11):** Пользователь отказывается. Бот благодарит и прощается. Перехода к другому задавателю нет — сессия завершается.
   - **Нечёткий ответ (узел 12, фоллбэк):** Бот просит уточнить, «да» или «нет», и возвращает вопрос: `[@goto("Ещё отмазку?")]`.

6. **Цикл основной генерации.** Каждый новый запрос пользователя при активном якоря `%that_anchor="Какая у тебя беда?"` пропускается через обработчики (узлы 3–6). Если ни один не сработал — срабатывает дерево-фоллбэк (`Какая у тебя беда? - *`, узел 7), который просит повторить запрос и возвращает к узлу-задавателю `[@goto("Какая у тебя беда?")]`.

7. **Глобальный fallback (`Fallback`, узел 8)** срабатывает, когда `%that_anchor` не установлен (или ни один активный якорь не подошёл) и ни один узел не поймал запрос. Приветствует пользователя, объясняет возможности и переводит в основной вопрос через `[@goto("Какая у тебя беда?")]`.

### Anti-repeat mechanism

- Anti-repeat обеспечивается встроенными механизмами `:lu:` (limit usage) и `[udict(name)]` на уровне ответов. Эти механизмы автоматически перебирают элементы словаря без повторов в рамках цикла/сессии.
- Переменные `%excuse_count` и `%last_excuse_style` НЕ используются.

### Глобальный fallback

- Узел `Fallback` в навыке `FALLBACK_GLOBAL` — единственная точка «универсального захвата» при отсутствии якоря. Глобальный fallback работает как `*` (wildcard) на самом низком уровне приоритета, без привязки к `%that_anchor`. CIAS (`%that_anchor="CIAS"`) используется только для EVENT-узлов.
