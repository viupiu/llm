# Бриф Оркестратора: Мини-Квест

## Метаданные
- **BotSlug:** `mini_quest_bot`
- **Канал:** чат (текстовый)
- **Режим оркестрации:** Batch
- **Дата:** 2026-05-29
- **Назначение:** Тестовый кейс для фабричного конвейера — проверка корректности построения диалогового графа, переходов, циклов и обработки тупиков

## Бизнес-задача
Создать развлекательный текстовый мини-квест, где пользователь проходит через серию выборов. Бот не требует сложной бизнес-логики, но требует **именно правильную структурную целостность** графа.

## Структура квеста
- **Время прохождения:** 3–5 минут
- **Узлы:** 10–15 (включая системные: EVENT Загрузка, Fallback)
- **Концовки:** ≥3 (успешные и неудачные)
- **Цикл:** ≥1 возвратный маршрут (ошибка → попытка пройти снова)
- **Граф:** без случайных тупиков, без битых ссылок

## Сценарий
Герой очнулся в таинственном подземелье. Перед ним — развилка, головоломка, выбор действий. В зависимости от решений — разные концовки.

### Поток
1. **EVENT Загрузка** → приветствие + описание ситуации → переход к первому вопросу
2. **Первая развилка:** «Пойти налево» / «Пойти направо» / «Осмотреться на месте»
3. **Вторая развилка** (зависит от выбора):
   - Наleft → дверь: «Открыть силой» / «Осмотреть механизм» / «Идти дальше влево»
   - Направо → река: «Перепрыгнуть» / «Найти мост»
   - Осмотреться → тайник: «Взять предмет» / «Игнорировать»
4. **Финальный выбор** с учётом собранных предметов/решений
5. **Концовки:**
   - «Свобода» (успех, правильный путь)
   - «Ловушка» (неудача, опасный выбор)
   - «Тупик» (неудача, не нашёл проход)

## Обязательные компоненты
- **EVENT Загрузки** — обязательный узел `EVENT 00b2fcbe-f27f-437b-a0d5-91072d840ed3` в навыке `EVENTS`
- **Fallback** — обязательный глобальный узел-звезда (`*`) для перехвата непредсказуемого ввода
- **Цикл** — хотя бы один узел должен вести пользователя обратно к предыдущему выбору при неудаче (например, «Перепрыгнуть» → «Падаете в воду» → возврат к «Найти мост»)
- **Якоря** — использовать паттерн задаватель/обработчик: узел задаёт вопрос, установливает `%that_anchor` → узлы-обработчики ловят по якорю

## Состояние (переменные)
- `%has_key` — есть ли ключ (взят/не взят)
- `%has_staff` — есть ли посох (взят/не взят)
- `%lives` — счётчик жизней (опционально, 1-2)
- `%that_anchor` — говорящие якоря, текст вопроса этапа

## Инфраструктура
- **EVENT:** `EVENT 00b2fcbe-f27f-437b-a0d5-91072d840ed3` (Загрузка)
- **CIAS:** для EVENT-узлов, `%that_anchor="CIAS"`
- **Fallback:** глобальный узел `*` без CIAS, возвращает к ближайшему активному якорю
- **Голосовые события:** не требуется (текстовый чат)

## Нужен ли ML-автор
Да. Узлов с реальным вводом пользователя >2, ML-автор необходим для генерации примеров классификации.

## Решения заказчика
- Нет IC/ML-обучения в реальной платформе — бот DL-first, ML-данные генерируются для полноты конвейера
- Без сложных счётчиков и внешних интеграций
- Без voice.end_call (текстовый бот)
- Квест не требует уникальности стиля — приоритет: структурная целостность графа

## Корректировки и отличия
- Граф должен содержать явный цикл (возвратный переход по `[@goto]`)
- Fallback должен возвращать к ближайшему узлу-задавателю через корректный переход
- Ни один пользовательский путь не должен уходить в тупик — каждая ветка заканчивается либо концовкой, либо циклом
