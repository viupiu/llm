## ОБЩАЯ UX-ЛОГИКА БОТА

### 1. Состояние настроения (%cat_mood)
Настроение — это «живой» атрибут кота. Оно влияет на ВСЕ ответы, делая их уникальными. Настроение не статичное — оно меняется в зависимости от действий пользователя:

| Действие | Новое настроение |
|----------|-----------------|
| Погладить | Довольный |
| Покормить | Довольный |
| Играть (довольный/игривый) | Довольный |
| Играть (голодный) | Довольный → затем Голодный |
| Играть (сонный) | Сонный |
| Играть (обиженный) | Обиженный |
| Похвалить | Довольный |
| Ругать | Обиженный |
| Игнорировать | Обиженный → Сонный |

### 2. Цикл взаимодействия
```
[Вход пользователя]
       │
       ▼
[Интерпретация узла] (Погладить, Покормить, Играть, Приветствие, Прощание, Благодарность, Угрозы, Fallback)
       │
       ▼
[Ответ кота + смена настроения]
       │
       ▼
[Возврат в «Мяу! Что будем делать?»]
       │
       └──→ (Цикл продолжается)
```

### 3. Рекурсия и разнообразие
- Каждое возвращение в главный узел — это новая фраза.
- Фразы не повторяются подряд.
- Настроение влияет на тон, длину и содержание фразы.
- «Довольный» — мурчащий, с благодарностью.
- «Голодный» — требовательный, с намёками на еду.
- «Сонный» — короткий, вялый, односложный.
- «Игривый» — энергичный, эмоциональный, многословный.
- «Обиженный» — холодный, односложный, с намёками на обиду.

### 4. Переносы строк ([br])
Все фразы содержат маркер [br] для обозначения переноса строки. При рендеринге заменяет на `\n`.

### 5. Звуковые элементы
- `Мяу`, `Мрр`, `Мур`, `Мур-мяу` — стандартные кошачьи звуки.
- Эти элементы можно анимировать: при нажатии на «Мяу» играет звук мяуканья.
- `Мррр` — мурчание, можно анимировать как вибрацию.
- Точки `...` — пауза, можно анимировать как задержку.

(Копирайтер:) Задача выполнена.
