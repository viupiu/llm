# 0_ORCHESTRATOR__BRIEF.md
# Бот: Подбор фильма на вечер
# BotSlug: movie_rec_bot

## 1. Общие сведения
- **Платформа**: чат
- **Режим оркестрации**: Interactive Mode
- **Краткое ТЗ**: Бот подбирает фильм на вечер, уточняя по жанру, настроению, компании, длительности и ограничениям.
- **Основная цель бота**: предложить пользователю фильм (пока заглушку-сценарий) на основе слотов, которые пользователь заполнит.

## 2. Сценарий и границы
Основные сценарии, которые должен покрывать бот:
- **Вход в сценарий**: жанр, настроение, компания (с кем смотреть), желаемая длительность.
- **Вход по отсылке**: «хочу что-то подобное на [фильм]».
- **Ограничения**: негативные предпочтения фильмов, которые пользователь не хочет видеть (скримеры, очень грустное).
- **Small Talk**: приветствия, благодарности, помощь.
- **Fallback**: возврат разговора пользователя с помощью fallback-узла в главный вопрос.

## 3. Возможные входные интенты (предварительно)
Следующие интенты могут быть сгенерированы ботом:
- `greeting` / `thanks` / `help`
- `recommendation_general` (общий запрос)
- `recommendation_genre` (запрос по жанру)
- `recommendation_mood` (запрос по настроению)
- `recommendation_similar` (запрос по похожему фильму)
- `recommendation_company` (запрос фильма по компании)
- `preference_duration` (предпочтительная длительность)
- `fallback`

## 4. Основные переменные состояния (slots)
- `genre` (жанр)
- `mood` (настроение)
- `company_type` (тип компании: с девушкой, с детьми, один и т.д.)
- `duration` (длительность: короткий, средний, длинный)
- `reference_movie` (название фильма-референса)
- `constraints` (ограничения: без скримеров, без грустного)

## 5. Ограничения контекста
- **Семантическая граница**: бот не является каталогом фильмов, он только собирает контекст пользователя.
- **Без ИИ**: никаких запросов к внешним API, подбора результатов из Интернета.
- **Неоднозначность**: допустимы неоднозначные интенты для заполнения слотов (ambiguous intent).

## 6. Технические требования
- **Проект**: Demo-scale (простой бот для теста всего pipeline).
- **ML-модель**: Не требуется (для теста генерации фраз).
- **Приоритеты**: Простая логика, отсутствие overengineering, корректные cross-file references.
