## 4. Критические замечания

### 🔴 Critical

**C-001: Отсутствует ML-датасет `4_EXAPTERS_AUTHOR__DATASET.md`**
- **Локация:** корень проекта `work/movie_rec_bot/`
- **Ошибка:** файл не создан
- **Требование:** для каждого узла с DL-правилами необходимо иметь не менее N обучающих примеров (согласно `D_GENERATION_RULES.md`)
- **Воздействие:** без ML-датасета бот не сможет обучаться на естественных формулировках пользователей; правила DL будут единственным источником матчинга
- **Действие:** вернуть автору датасета для генерации примеров

### 🟠 Major

**M-001: Опечатка в правиле «Настроение» (дерево 1)**
- **Локация:** `3_RULES_AUTHOR`, строка 59
- **Ошибка:** `чуеств~` вместо `чувств~`
- **Требование:** точные стеммы в синтаксисе правил
- **Исправление:** заменить `чуеств~` на `чувств~`

**M-002: Рассинхрон текста ответа и логики перехода в узле «Компания» (дерево 1)**
- **Локация:** `6_RESPONSES_AUTHOR`, строки 122-123
- **Ошибка:** текст ответа спрашивает «А по длительности что предпочтешь?», но `[%that_anchor="Какой жанр вам по душе?"]` — следующий вопрос о жанре
- **Требование:** текст ответа должен соответствовать следующему вопросу
- **Исправление:** заменить текст на "Какой жанр тебе ближе?" или пересмотреть логику переходов
- **Примечание:** архитектура specifies переход к жанру (slot-filling из ранних ответов), но это контринтуитивно для UX

**M-003: Избыточный inline-список в правиле «Ограничения» (дерево 1)**
- **Локация:** `3_RULES_AUTHOR`, строка 92
- **Ошибка:** `{н~/н~/н~/н~/н~/н~/~не~/~не~/~без/~без/~без/~не хочу/~не хочу/~не хочу}` — 18 элементов, 5 уникальных паттернов
- **Требование:** уникальные элементы в inline-списках
- **Исправление:** `{н~/~не~/~без/~не хочу}` — 4 элемента
- **Риск:** степп `н~` без контекста матчит огромное количество слов, повышая риск false positives

### 🟡 Minor
