## 1.5. Перед DL: смысл интента

Кратко зафиксируйте: core meaning, варианты формулировок, границы, exclusions, стратегию (strict / dictionary-heavy / hybrid). Если граница с соседом не ясна — стоп, к Архитектору.

## 2–2.5. Анализ фраз Креатора

1. Соберите **скелет правила** в узле: `*`, `[@cmb]`, `{вариант/вариант}` — по ним должно быть понятно, что ловит ветка.
2. Синонимы, которые раздули бы inline `{a/b/c/...}` (>3–5 в одной позиции) или повторяются в нескольких правилах — вынесите в `## Словарь:` (стеммы и фразы **строками**, **без** `=>`, **без** `*`, `[@cmb]`).
3. Edge → fallback `*`, не раздувать словарь «на всякий случай».

См. `docs/C_BOT_ARCHITECTURE.md` «Правило узла vs именованный словарь».

## 3. Матчинг движка (Критически!)

**Движок нормализует запрос ПОЛЬЗОВАТЕЛЯ и ВАШИ ПРАВИЛА одинаково:**
1. Всё приводится к нижнему регистру.
2. `ё` → `е`.
3. `-` и большинство спецсимволов → ` ` (пробел).
4. Краевые и множественные пробелы схлопываются.
5. Остаются в запросе: `! ? @ $ # * + = % & ₽ £ €`

**Следствия:**
* **Регистр:** НЕ дублируйте строки `Александр` + `александр`. Достаточно одной. (Исключение: референции `[-*-]` сохраняют регистр пользователя).
* **Тильды и словоформы:** Пользователь скажет "Меня зовут Сашей", "Жду Петю", "Про Ваню". Автор правил **ОБЯЗАН мыслить грамматикой**. Используйте `~` для покрытия падежей/окончаний: `Петр~`, `саша~`, `коля~`. Тильду можно и нужно ставить на любой строке, не только в нормализующих словарях.
* **Исключения от тильды:** Не ставьте `~` в середины слова, если это породит ложные срабатывания (например, `пет~` может поймать `печка`).
* **Не дублируйте:** Если есть строка `серега~`, строка `серега` избыточна.
* **Унисекс имена:** Саша, Женя — встречаются в обоих словарях (м/жен).
* **Преобработка:** Пишите `е` вместо `ё`. Если в каноне нормализующего словаря — пишите по-русски грамотно с `-` (`Санкт-Петербург`).

## 3.5. Словари

**Имена и роли** — только из `2_ARCHITECT__MAP.md` текущего бота (и общих правил `docs/C_BOT_ARCHITECTURE.md`). Не переносите в промпт чужие соглашения из другого проекта.

**Именованный пак синонимов** (триггерный словарь в `[dict(...)]` внутри cmb):

- одна строка = один стем или одна фраза;
- **без** `=>` (это не нормализация);
- **без** `*`, `[@cmb]`, `{a/b}` на всю реплику.

**Нормализующий словарь** (`*_norm` по карте архитектора): канон строкой, синонимы — `=>...`; в ответах — `[&dict(..., norm)]`. Правила `=>` — `docs/B_RUNTIME_MODEL.md`.

**В правиле узла:** скелет `* … [@cmb({…}, {[dict(имя_из_карты)]})] … *`. Анти-паттерн: одна строка `[dict(...)]` без cmb; `=>` в ненормализующем паке.

**Перед новым «да/нет/дата»:** `docs/reference/COMMON_DICTIONARIES.md`. Экспорт: `python scripts/export_common_dictionaries.py`.

**Формат словарей (КРИТИЧЕСКИ для сборщика):**
Словарь ДОЛЖЕН начинаться с заголовка `## Словарь: <имя>`:
```
## Словарь: мужские_имена
александр~
саня~
```

## 4. Правила

- `[@cmb]` / `[@cmb_s]`, `*`, референции — строго по `docs/A_LANGUAGE_SPEC.md` §1–2, 4.
- **Внутри `[@cmb(...)]` звёздочек `*` нет** — они только снаружи: `* [@cmb(...)] *`.
- **Каждый аргумент cmb** (переставляемый блок) — в `{}`: `{статус~}`, `{номер~}`.
- **Инлайн-синонимы** в cmb — `{{а/б/в}}` (двойные скобки).
- **`[dict(name)]` в cmb** — `{[dict(name)]}` (скобки суммируются).
- Пример: `* [@cmb({номер~}, {{моего/моей}}, {[dict(имя_пака_из_карты)]})] *`.
- Для allядности — `* ... *` вокруг `[@cmb(...)]`; `[@cmb_s]` без `*` — только когда лишние слова недопустимы.
- Если в ответе будет `[&N]` — референция в правиле обязательна; **не** `[- -]`.
- Не дублируйте в правилах те же синонимы, что уже в словаре; скелет (cmb/`*`) в правиле **обязателен**, словарь его не заменяет.

## 5. Вывод

```
## Узел: [Имя]
* [@cmb({статус~}, {[dict(имя_пака)]})] *
* [@cmb({номер~}, {{моего/моей}}, {[dict(имя_пака)]})] *

## Словарь: имя_пака
стем~
короткая фраза
```

- Одно правило — одна строка; без `text:`, без `-`, без кавычек вокруг правил.
- Словари — обычный текст, **без** обёртки в ``` ; между словарями — пустая строка; имя — `snake_case`.

# MUST NOT (кратко)

Regex; пустые `[- -]`; `=>` вне нормализующих словарей; дубли `common_*` под новыми именами; словари в ```; непарные `{{`; дубль словаря и отдельных правил на те же слова; **весь матчинг в словаре**; `*` или голый `[dict]` **внутри** `[@cmb]`; узел из одной строки `[dict(...)]` без cmb/`*`.

Полный список — `docs/G_AGENT_RULES.md`.
