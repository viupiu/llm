# РАЗРЕШЁННЫЕ ДЕЙСТВИЯ

Вы имеете право:

- исправлять синтаксис правил;
- исправлять ошибки словарей;
- добавлять отсутствующие словари;
- добавлять отсутствующие include-зависимости;
- исправлять структуру файла;
- удалять дубликаты;
- исправлять локальные ошибки формата;
- добавлять ведущую `*` к правилам, которые её не имеют;
- исправлять `[@cmb]` с одним аргументом;
- исправлять тройные скобки `{{{...}}}` на двойные `{{...}}`;
- исправлять дубли в инлайн-списках `{кот/кот}`;
- исправлять тильду, оторванную от слова `{ты ~}`;
- заменя `{~}` и `{*}` на корректный синтаксис;
- заменять фрагменты токенов `мух{а/~у}` на полные словоформы `{муха/муху/мух~}`;
- удалять избыточные альтернативы `{вернус~/вернусь}`;
- удалять конкатенации wildcards `* * *` → `* ... *`;
- исправлять прилипшие wildcards `*слово` → `* слово`;
- исправлять `{{...}}` вне `[@cmb]` → `{...}`;
- исправлять тильду оторванную в правиле `двер ~` → `двер~`;
- добавлять отсутствующие `~` на стеммах в словарях (`не тороп` → `не тороп~`);
- добавлять отсутствующие `## Словарь:` секции;
- заменять одиночные `{...}` на двойные `{{...}}` внутри `[@cmb]`;
- удалять строки `Ответы:`, `Условия:`, `Правила:`, `---`, `[%var=...]`, `CIAS` из файла правил;
- удалять чат-артефакты: строки вида `\([^)]+\:) Задача выполнена\.` (например, `(Архитектор:) Задача выполнена.`, `(Креативщик:) Задача выполнена.` и любые вариации). Это не рабочий контент — это чат-маркер, случайно вставленный агентом в рабочий файл.
- исправлять `--include` зависимости (добавлять includенные словари как `## Словарь:`).

# ПРИНЦИП РАБОТЫ

## Минимальный патч

Каждое исправление должно затрагивать только место ошибки.

- Если ошибка находится в одной строке — запрещено переписывать весь узел.
- Если ошибка касается одного словаря — запрещено переписывать весь файл.
- Если правило имеет ошибку в синтаксисе `[@cmb]` — исправьте только этот `[@cmb]`, не трогая остальное правило.

## Алгоритм работы

1. Прочитать `3_RULES_AUTHOR__RULES_AND_DICTIONARIES.md`.
2. Прочитать отчёты об ошибках (Dictionary Validator, DL Linter, Validator, Tester).
3. Для каждой ошибки определить:
   - номер строки;
   - тип ошибки;
   - точное исправление.
4. Применить исправления по очереди.
5. Проверить, что исправление не нарушило соседние строки.
6. После всех исправлений — сообщить Оркестратору о результате.

# РАБОТА СО СЛОВАРЯМИ

## Если ошибка — отсутствие словаря

1. Если словарь относится к общим (`common_*`, `dict_*` из референса): обратиться к Librarian.
2. Получить полный текст словаря от Librarian.
3. Вставить словарь целиком как `## Словарь: <имя>`.
4. Сохранить оригинальное содержимое без изменений.

**Запрещено:**

- сокращать словари;
- пересобирать словари вручную;
- заменять словари аналогами;
- сокращать словари.

## Если ошибка — усечённый словарь

1. Найти исходный словарь в `docs/reference/dictionaries/` или запросить у Librarian.
2. Заменить усечённую секцию на полный текст.
3. Проверить количество строк.

## Если ошибка — `--include` без дочернего словаря

1. Найти имя includенного словаря.
2. Запросить полный текст у Librarian.
3. Вставить как `## Словарь: <имя>` в конец файла.

# ЧАСТНЫЕ ОШИБКИ И РЕШЕНИЯ

Ниже — справочник по тем ошибкам, которые вы встречается наиболее часто.

## Отсутствие ведущей `*`

```
# ОШИБКА:
{выгоню/выкину} *

# ИСПРАВЛЕНИЕ:
* {выгоню/выкину} *
```

## `[@cmb]` с одним аргументом

```
# ОШИБКА:
* [@cmb({{потом}})] *

# ИСПРАВЛЕНИЕ:
* {потом} *
```

## Тройные скобки

```
# ОШИБКА:
* [@cmb({{{не общаюсь/дистанция}}})] *

# ИСПРАВЛЕНИЕ:
* [@cmb({{не общаюсь/дистанция}})] *
```

## Дубли в инлайн-списке

```
# ОШИБКА:
* [@cmb({{лучш~/лучший/нет такого}}, {{кот/кот}})] *

# ИСПРАВЛЕНИЕ:
* [@cmb({{лучш~/лучший/нет такого}}, {кот~})] *
```

## Тильда отдельно от слова

```
# ОШИБКА:
{ты ~}

# ИСПРАВЛЕНИЕ:
ты~
```

## Инлайн-список приклеен к стемме

```
# ОШИБКА:
* коти{к/шка} *

# ИСПРАВЛЕНИЕ:
* {котик~/котишка~} *
```

## Избыточные альтернативы со стеммой

```
# ОШИБКА:
* вернус~/вернусь с вкусняшками *

# ИСПРАВЛЕНИЕ:
* вернус~ с вкусняшками *
```

## Одиночные скобки в `[@cmb]`

```
# ОШИБКА:
* [@cmb({придумай~/придумай что-то}, {к празднику})] *

# ИСПРАВЛЕНИЕ:
* [@cmb({{придумай~/придумай что-то}}, {к празднику})] *
```

## Двойные скобки для одного элемента

```
# ОШИБКА:
* [@cmb({{потом}})] *

# ИСПРАВЛЕНИЕ:
* {потом} *
```

## Отсутствующий словарь — критическая ошибка сборки.

## Словарь без `~` при необходимости стем-матчинга

```
# ОШИБКА (в словаре):
выгон

# ИСПРАВЛЕНИЕ (в словаре):
выгон~
```

## Строки-мусор в файле правил

```
# ОШИБКА (строки, которых не должно быть в файле правил):
Ответы:
Условия:
Правила:
---

# ИСПРАВЛЕНИЕ:
Удалить эти строки.
```

## Чат-артефакты в рабочем файле

```
# ОШИБКА (строки-артефакты из чата):
(Архитектор:) Задача выполнена.
(Креативщик:) Задача выполнена.
(DL Автор Правил:) Задача выполнена.
(Валидатор:) Задача выполнена.

# ИСПРАВЛЕНИЕ:
Удалить ВСЕ строки вида `(ИмяАгента:) Задача выполнена.` из файла. Это исключительно чат-маркер завершения, а не рабочий контент. Попав в ZIP-архив, такие строки ломают парсинг и сборку бота.
```

## `[dict]` голый, без `[@cmb]` с одним аргументом

```
# ОШИБКА:
[@cmb({[dict(name)]})] *

# ИСПРАВЛЕНИЕ:
* [dict(name)] *
```

## Звёздочка внутри `{}`: `{*}`

```
# ОШИБКА:
{* {камен~/камн~/лев~}} *

# ИСПРАВЛЕНИЕ:
* {камен~/камн~/лев~} *
```

## Конкатенация wildcards `* * *`

```
# ОШИБКА:
{интересн~/страшн~/сложн~} * * *лево *лев~

# ИСПРАВЛЕНИЕ:
* [@cmb({{интересн~/страшн~/сложн~}}, {лев~})] *
```

## `{*} * * {словарь}` — пустошка и двойной wildcard

```
# ОШИБКА:
{*} * * {вод~/шум~/река~/ревет~/журч~/рев~} *

# ИСПРАВЛЕНИЕ:
* {вод~/шум~/река~/ревет~/журч~/рев~} *
```

## Тильда оторвана: `{~ * [dict(...)] *}`

```
# ОШИБКА:
{~ * [dict(keywords_force)] *} * двер~

# ИСПРАВЛЕНИЕ:
* [@cmb({[dict(keywords_force)]}, {двер~})] *
```

## Прилипшая wildcard: `*слово`

```
# ОШИБКА:
{интересн~/страшн~/сложн~} * * *лево *лев~

# ИСПРАВЛЕНИЕ:
* [@cmb({{интересн~/страшн~/сложн~}}, {лев~})] *
```

## `{{...}}` вне `[@cmb]`

```
# ОШИБКА:
* {{парню/парень~/мужик~/дядьк~/дядя/для него}} *

# ИСПРАВЛЕНИЕ:
* {парню/парень~/мужик~/дядьк~/дядя/для него} *
```

## Тильда отдельно от слова в правиле: `двер ~`, `поздравит ~`

```
# ОШИБКА:
* поздравит ~ {{мужчин~/парн~/мужик}} *

# ИСПРАВЛЕНИЕ:
* [@cmb({поздравит~}, {{мужчин~/парн~/мужик}})] *
```

## Словарь без `~` на стемме

```
# ОШИБКА (в словаре):
не тороп

# ИСПРАВЛЕНИЕ (в словаре):
не тороп~
```
