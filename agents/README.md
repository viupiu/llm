# LLM-Фабрика — Многоагентный конвейер разработки ботов

Автоматизированный пайплайн для создания чат-ботов и голосовых обзвонщиков на платформе ДОС (Dialogflow-like). Каждый узел диалога проходит через серию специализированных агентов.

---

## Схема конвейера

```
Пользователь → ТЗ
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│  0.  Оркестратор ── бриф ── → 1.  Архитектор ── карта ──→       │
│   (менеджер)     (blueprint)         (узлы, переходы,           │
│                                     словари, интенты)           │
│                                              │                  │
│                                              ▼                  │
│                                   ┌────────────────────┐        │
│                                   │  2.  Креативщик    │        │
│                                   │ (фразы пользователя│        │
│                                   │  для каждого узла) │        │
│                                   └────────┬───────────┘        │
│                                            │                    │
│                           ┌────────────────┼         			  │
│                           ▼                ▼                    │
│                    3. Автор Правил   4. Автор                   │
│                    (DL-правила       Примеров                   │
│         ┼───────── + словари)      (ML-данные)                  │
│         ▼                                │                      │
│   3.5 Тестер              5. Копирайтер ◄┘                      │
│  (проверка DL-покрытия)     (UX-тексты)                         │
│                               │                                 │
│                               ▼                                 │
│                        6. Автор Ответов                         │
│                      (DL-ответы + условия)                      │
│                               │                                 │
│                               ▼                                 │
│                        7. Валидатор                             │
│                        (финальный аудит)                        │
│                               │                                 │
│                               ▼                                 │
│                        8. Упаковщик ──→ ZIP для импорта         │
└─────────────────────────────────────────────────────────────────┘

 Сквозные агенты:
 ─ Менеджер Глоссария (0.5) — контроль терминологии на любом этапе
 ─ Координатор (9) — навигация по файловой структуре
 ─ Сборщик тестового сета (99) — формирование CSV для ручных тестов
```

### Краткие роли

- **Оркестратор** — центральный менеджер: собирает ТЗ, формирует бриф, контролирует поток, разрешает конфликты, обновляет память проекта.
- **Архитектор** — проектирует скелет бота: узлы, переходы, интенты, словари, состояния.
- **Креативщик** — генерирует 200–300 реалистичных фраз пользователя на каждый узел.
- **DL Автор Правил** — превращает фразы в DL-правила (`dl_rules`) и словари с полным покрытием без перехвата соседних узлов.
- **ML Автор Примеров** — отбирает ~100 эталонных примеров на узел для обучения классификатора интентов.
- **Копирайтер** — пишет живые UX-тексты ответов бота, описывает поведение, память, переходы.
- **DL Автор Ответов** — компилирует тексты в DSL-ответы (`dl_answers`), условия, goto-переходы, сохранение переменных.
- **Валидатор** — финальный бескомпромиссный аудит: синтаксис DL, целостность, покрытие всех узлов.
- **Упаковщик** — запускает Python-скрипты для сборки `staging/` и ZIP-архива для импорта в платформу.
- **Библиотекарь** — семантический куратор общих словарей: анализирует содержимое `COMMON_DICTIONARIES.md`, выявляет дубликаты, misleading-имена, updating описания

## Ключевые правила

- **Узкая специализация.** Каждый агент — только своя зона ответственности.
- **Строгая очередность.** Downstream-агент не запускается на `stale`/`rejected` артефактах.
- **Пауза.** Оркестратор запрашивает подтверждение перед следующим этапом.
- **Идентификация в чате.** Каждый фрагмент ответа начинается с префикса `(ИмяАгента:)`.
- **Завершение задачи.** Последний фрагмент: `(ИмяАгента:) Задача выполнена.`
- **Обязательная запись уроков.** Любое замечание пользователя → запись в файл (через Координатора). См. `docs/H_USER_INTERACTION_PROTOCOL.md`.
- **Русский язык.** Все артефакты, комментарии, тексты — строго на русском.
- **Русский язык коммитов.** Текст коммита — **ТОЛЬКО на русском языке**. MUST NOT писать commit message на английском. Формат: `{описание на русском} (Оркестратор)`. См. `docs/G_AGENT_RULES.md` §11 + инцидент 2026-05-28.
