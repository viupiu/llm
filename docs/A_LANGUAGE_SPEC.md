# A. СИНТАКСИС ЯЗЫКА DL (LANGUAGE_SPEC)

**SSOT для:** Профиль проекта, синтаксис поисковых правил, словари, референции, переменные, условия, функции, таблицы, комментарии, внешние действия.

**НЕ содержит:** Логику выполнения (ссылка → B), Архитектуру бота (ссылка → C), Форматирование ответов (ссылка → E), API платформы (ссылка → F).

---

## Профиль проекта

| Тема | Правило |
|------|---------|
| Уникальность ответов | Используем `:u:` и `:lu:`. Модификаторы `:uniq:` / `:luniq:` из полного описания DOS **не входят** в проектный профиль. |
| Include в словарях | Только `--include(имя_словаря)` — **ровно два дефиса** в начале. Варианты `–include` (длинное тире) недопустимы. |
| Имена переменных | Допустимы латиница, кириллица, цифры, `_`, `-`, `.` (как в DOS). В ответах и условиях: `[%имя]`, в `conditions`: `%имя="значение"`. |

Примеры имён переменных: `%user_name`, `%этап-1`, `%voice.end_call`.

| Конструкция | Значение |
|---|---|
| `слово` | точное совпадение |

| `{a/b/c}` | inline-список |
| `{a/b/--}` | inline-список с пустым вариантом |
| `<!>` | факультативный элемент |

```dl

{да/ага/угу}
```

## 2. Комбинаторика

### `[@cmb]` — Любой порядок, лишние слова разрешены

```dl
[@cmb({оплатить}, {счет})]
```

**Скобки аргументов `[@cmb]` (обязательно):**

| Элемент | Запись | Пример |
|---------|--------|--------|
| Один термин / стем | `{слово~}` | `{статус~}`, `{номер~}` |
| Инлайн-синонимы | `{{а/б/в}}` | `{{моего/моей}}` |
| Именованный словарь | `{[dict(имя)]}` | `{[dict(keywords_appeal)]}` |
| Звёздочки `*` | **только снаружи** | `* [@cmb({статус~}, {[dict(keywords_appeal)]})] *` |

**Запрещено внутри `[@cmb(...)]`:** `* статус *`, голый `[dict(name)]` без `{}`, голый `статус~` без `{}`.

### `[@cmb_s]` — Любой порядок, лишние слова запрещены

```dl
[@cmb_s({да}, {конечно})]
```

### `[@cmb_n]` — Разрешает ровно N слов между элементами

```dl
[@cmb_n(1, {хочу}, {кредит})]
```

## 3. Словари

### Использование

```dl
[dict(city)]
[lcdict(city)]
```

### Формат словаря

```text
Москва
=>москв~
=>мск

Санкт-Петербург
=>питер~
=>спб
```

### Правила словарей
- Канон всегда без `~`
- `=>` только в нормализующих словарях
- Словарь не может состоять только из `=>`

### Include: `--include`

В теле словаря отдельной строкой:
```text
--include(other_dict)
```
- Строго с двумя ASCII-дефисами: `--include`
- Не дублируйте содержимое вручную, если достаточно include базового словаря

## 19. Общие правила синтаксиса

- Парность скобок: `[` = `]`, `{` = `}`, `(` = `)`
- НЕ использовать `=>` в ненормализующих словарях
- `[if(...)]{...}` — без пробела между `)` и `{`
- `[dict]` в ответах: если словарь содержит `=>` (нормализация), использовать `[&dict(name, norm)]`
- `[&1]` — только если в `dl_rules` есть `[- -]` или `[-{...}-]`
- `[@hvs("table", "row")]` или `[@hvs("table", "row", "col")]`

## Правила из dl_author_rules.md §4

- `[@cmb]` / `[@cmb_s]`, `*`, референции — строго по этим разделам
- **Внутри `[@cmb(...)]` звёздочек `*` нет** — они только снаружи
- **Каждый аргумент cmb** — в `{}`: `{статус~}`, `{номер~}`
- **Инлайн-синонимы** в cmb — `{{а/б/в}}`
- **`[dict(name)]` в cmb** — `{[dict(name)]}`
- Если в ответе будет `[&N]` — референция в правиле обязательна
- Не дублируйте синонимы из словаря в правила; скелет (`*`, `[@cmb]`) в правиле **обязателен**

## 21. Комбинаторный метод и атомарность

**Комбинаторный метод (`[@cmb]`):** Вместо перечисления вариантов — облако смысловых элементов в `[@cmb]`. Движок сам генерирует перестановки.

**Атомарность словарей:** Общие словари используются в нескольких узлах для синхронного обновления лексикона. Один словарь — синхронная правка везде.

---

## 22. Стемминг: тильда vs словоформы

- `~` — вариативное окончание. `кот~` → ловит `кота`, `коту`, `кого-то`.
- **Осторожно:** `кот~` может ловить лишнее (например, `когда-то`). Для критичных слов используйте `{кот/коту/котом}`.
- **Словари словоформ:** отдельный пакет с префиксом `word_` (например `word_cat`), содержащий все словоформы вручную. Используйте когда `~` недостаточно точен.

## 21. Комбинаторный метод (`[@cmb]`)

Вместо явного перечисления всех формулировок — облако смыслов. `[@cmb]` генерирует все перестановки позиций, позволяя элементам появляться в любом порядке с произвольным количеством слов между ними.

**Принцип:** выделите ключевые слова интента → заверните в `[@cmb({ключ1}, {ключ2}, {[dict(пак)]})]`.

**Атомарность:** общие словари используются в нескольких узлах для синхронного обновления лексикона.

---

## 22. Стемминг и словари словоформ

- **Префикс `word_`:** словари, содержащие полную таблицу словоформ (падежи, числа). Пример: `word_cat` = `кот`, `кота`, `коту`, `коте`, `котом`, `коты`, `кошек`...
- **Стемминг `~` vs словоформы:** `~` покрывает вариативные окончания автоматически, но может ловить лишнее. Для критичных слов используйте `{кот/коту/котом}` или `word_`-словарь.
