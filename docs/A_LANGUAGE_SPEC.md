# A. СИНТАКСИС ЯЗЫКА DL (LANGUAGE_SPEC)

**SSOT для:** Профиль проекта, синтаксис поисковых правил, словари, референции, переменные, условия, функции, таблицы, комментарии, внешние действия.

**НЕ содержит:** Логику выполнения (ссылка → B), Архитектуру бота (ссылка → C), Форматирование ответов (ссылка → E), API платформы (ссылка → F).

## 1. Профиль проекта

| Тема | Правило |
|------|---------|
| Уникальность ответов | Используем `:u:` и `:lu:`. Модификаторы `:uniq:` / `:luniq:` из полного описания DOS **не входят** в проектный профиль. |
| Include в словарях | Только `--include(имя_словаря)` — **ровно два дефиса** в начале. Варианты `–include` (длинное тире) недопустимы. |
| Имена переменных | Допустимы латиница, кириллица, цифры, `_`, `-`, `.` (как в DOS). В ответах и условиях: `[%имя]`, в `conditions`: `%имя="значение"`. |

Примеры имён переменных: `%user_name`, `%этап_1`, `%voice.end_call`.

## 3. Комбинаторика

### `[@cmb]` — Любой порядок, лишние слова разрешены

```dl
[@cmb({оплатить}, {счет})]
```

**Скобки аргументов `[@cmb]` (обязательно):**

| Элемент | Запись | Пример |
|---------|--------|--------|
| Один термин / стем | `{слово~}` | `{~ехать}`, `{~регистр~}` |
| Инлайн-синонимы | `{{а/б/в}}` | `{{моего/моей}}` |
| Именованный словарь | `{[dict(имя)]}` | `{[dict(keywords_appeal)]}` |
| Звёздочки `*` | **только снаружи** | `* [@cmb({статус~}, {[dict(keywords_appeal)]})] *` |

**Запрещено внутри `[@cmb(...)]`:** голый `[dict(name)]` без `{}`, голый `статус~` без `{}`.

**Разница между `{...}` и `{{...}}`:**
| Контекст | Синтаксис | Назначение |
|----------|-----------|------------|
| Вне `[@cmb]` (в `*...*`) | `{а/б/в}` | Инлайн-список синонимов |
| Внутри `[@cmb]` (один элемент) | `{стем~}` | Один стем или один словарь |
| Внутри `[@cmb]` (синонимы) | `{{а/б/в}}` | Инлайн-синонимы (обязательно двойные скобки) |

- **Внутри `[@cmb(...)]` звёздочек `*` нет** — они только снаружи
- **Каждый аргумент cmb** — в `{}`: `{статус~}`, `{номер~}`
- **Инлайн-синонимы в `[@cmb]` — ОБЯЗАТЕЛЬНО в двойных скобках `{{а/б/в}}`.** Одиночные `{а/б/в}` внутри `[@cmb]` запрещены!
- **`[dict(name)]` в cmb** — `{[dict(name)]}`

### `[@cmb_s]` — Любой порядок, лишние слова запрещены

```dl
[@cmb_s({да}, {конечно})]
```

### `[@cmb_n]` — Разрешает ровно N слов между элементами

```dl
[@cmb_n(1, {хочу}, {кредит})]
```

### Методологические заметки: Комбинаторный метод

**Комбинаторный метод (`[@cmb]`):** Вместо явного перечисления всех формулировок — облако смысловых элементов в `[@cmb]`. Движок сам генерирует все перестановки позиций, позволяя элементам появляться в любом порядке с произвольным количеством слов между ними.

**Принцип:** выделите ключевые слова интента → заверните в `[@cmb({ключ1}, {ключ2}, {[dict(пак)]})]`.

## 4. Словари

### Использование

```dl
[dict(city)]
[lcdict(city)]
```

### Формат словаря

```text
Москва
=>москв~
=>мск

Санкт-Петербург
=>питер~
=>спб
```

### Правила словарей
- Канон всегда без `~` и `*`
- `=>` только в нормализующих словарях
- Словарь не может состоять только из строк, начинающихся с `=>`

### Include: `--include`

В теле словаря отдельной строкой:
```text
--include(other_dict)
```
- Строго с двумя ASCII-дефисами: `--include`
- Не дублируйте содержимое вручную, если достаточно include базового словаря

### Методологические заметки: Атомарность словарей

**Атомарность:** Общие словари используются в нескольких узлах для синхронного обновления лексикона. Один словарь — синхронная правка везде. Не дублируйте синонимы из словаря в правила; скелет (`*`, `[@cmb]`) в правиле **обязателен**.

## 5. Референции

### В правилах
```dl
[-*-]
[-слово-]
[-{вариант1/вариант2}-]
[-[dict(name)]-]
```

### В ответах
```dl
[&1]
[&dict(city, norm)]
```
В конструкции `[&1]` число означает порядковый номер референции в строке правила.
Правило: `[-{я/ты}-] [-{умный/дурак}-]`
В ответе: `[&1][&2]`
Если пользователь скажет: `я умный`, то в ответе: `[&1]`=`я`, `[&2]`=`умный`
- Если в ответе будет `[&N]` — референция в правиле обязательна

## 6. Переменные и сущности

### Переменные
```dl
[%user_name]
[%user_name="Вася"]
[%user_name=""]
[%этап_1]
[%voice.end_call="True"]
```

**Имя переменной:** латиница, кириллица, цифры, `_`, `.`
В `conditions` — без скобок: `%that_anchor="значение"`.

### Сущности
```dl
[$city]
[$tariff]
```

### Сохранение сущности
```dl
[%city="[$city]"]
```

### Синтаксис присвоения переменных

| Конструкция | Описание |
|---|---|
| `[%name]` | Вывод |
| `[%name="значение"]` | Присвоение |
| `[%name=""]` | Сброс |
| `[%name="[$entity]"]` | Сохранение сущности ✅ |
| `[%name=$entity]` | Неправильно ❌ |

## 10. Таблицы

### `@hvs`
Ответ будет выдан, даже если ячейка таблицы пустая.
```dl
[@hvs("table", "row")]
[@hvs("table", "row", "column")]
[@hvs("messengers", "[&dict(messenger, norm)]")]
```

### `@shvs`
Если ячейка таблицы пустая, движок найдет другой ответ на запрос пользователя.
```dl
[@shvs("table", "row")]
[@shvs("table", "row", "column")]
```

## 14. Общие правила синтаксиса

- Парность скобок: `[` = `]`, `{` = `}`, `(` = `)`, `"` = `"`
- НЕ использовать `=>` в ненормализующих словарях
- `[if(...)]{...}` — без пробела между `)` и `{`
- `[dict]` в ответах: если словарь содержит `=>` (нормализация), если есть задача нормализовать вхождение, использовать `[&dict(name, norm)]`
- `[&1]` — только если в `dl_rules` есть `[- -]` или `[-{...}-]`
- `[@hvs("table", "row")]` или `[@hvs("table", "row", "col")]` или `[@shvs("table", "row")]` или `[@shvs("table", "row", "col")]`

### Синтаксические анти-паттерны (КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО)

| Антпаттерн | Правильно | Причина |
|------------|-----------|---------|
| `{общ/~}` — тильда как самостоятельный элемент инлайн-списка | `общ~` или `{общ~/--}` | Тильда — модификатор стеммы, не самостоятельный элемент DSL |
| `[@cmb({а/б}, {в})]` — одиночные скобки для синонимов в cmb | `[@cmb({{а/б}}, {в})]` или `[@cmb({{а/б}}, {{в/г}})]` | Списки синонимов в cmb только в `{{...}}` |
| `[@cmb({{стем1/стем2}}, {{один}})]` — двойные `{{}}` для ОДНОГО элемента | `[@cmb({{стем1/стем2}}, {один})]` | Двойные `{{}}` — только для синонимов; один элемент = `{один}`. Тройные `{{{...}}}` — ЗАПРЕЩЕНО. |
| `{ты ~}` — тильда оторвана от слова пробелом | `ты~`, `~ты` | Тильда должна быть приклеена к символу слова |
| `мух{а/ааа/~у}` — `{...}` «приклеен» к префиксу; элементы `{...}` — неполные токены; `~` модифицирует фрагмент | `{муха/муху/мух~}` | `{...}` — отдельный токен. `~у` = любое слово на `у`, экстремально широкое совпадение. Элементы инлайн-списка всегда самодостаточные. См. D §16.7. |
| `[dict(name)] *` — пропущена ведущая `*` | `* [dict(name)] *` | Правило всегда начинается и заканчивается `*` |
| `---` в конце словаря | Нет маркера | Словари не содержат разделителей-линий |

### Анти-паттерны `[@cmb]` (Инцидент 2026-05-29, Cat Simulator)

> **ВНИМАНИЕ:** ниже приведены реальные ошибки, выявленные при распаковке `cat_simulator.zip`.

| Ошибка (REDACTED из prod) | Правильно | Причина |
|---------------------------|-----------|---------|
| `* [@cmb({потом}, {поиграем/поболтаем/приду})] *` | `* [@cmb({потом}, {{поиграем/поболтаем/приду}})] *` | `{поиграем/поболтаем/приду}` — это синонимы, они ОБЯЗАНЫ быть в `{{...}}`. Одиночные `{...}` внутри `[@cmb]` допускаются ТОЛЬКО для одного элемента: `{потом}`. |
| `* [@cmb({{хочу/руки тянутся/невозможно}}, {{поглад~}})] *` | `* [@cmb({{хочу/руки тянутся/невозможно}}, {поглад~})] *` | `{поглад~}` — один элемент, не синонимы. Двойные `{{...}}` для одного элемента — ОШИБКА. |
| `* [@cmb([dict(name)])] *` | `* [dict(name)] *` | `[@cmb]` с одним аргументом — ЗАПРЕЩЕНО. Один словарь = пишем напрямую. |
| `* [@cmb({статус~}, *слово*)] *` | `* [@cmb({статус~}, {слово~})] *` | Звёздочки `*` внутри `[@cmb(...)]` — ЗАПРЕЩЕНО. `*` только снаружи. |

**Правило быстрого определения скобок в `[@cmb]`:**
1. Один элемент → `{один}`.
2. Два и более синонима → `{{синоним1/синоним2}}`.
3. Словарь → `{[dict(name)]}`.
4. Никогда не пишите `{{{...}}}`.

## 15. Стемминг и словоформы

- `~` — вариативное окончание или начало слова. `кот~` → ловит `кота`, `коту`
- **КРИТИЧЕСКАЯ ОГРАНИЧЕНИЕ:** `~` ДОЛЖЕН стоять ТОЛЬКО в начале или в конце токена. `~` в середине слова (например, `мужик~у`, `слов~арь`) — КАТЕГОРИЧЕСКИ ЗАПРЕЩЁН.
- **Принцип "Минимальный уникальный корень":** Стем должен быть максимально коротким, чтобы покрывать все нужные словоформы. Если `поздрав~` покрывает `поздравит`, `поздравление`, `поздравлень` — писать `поздравлень~` ОШИБКА. Всегда берите самую короткую стемму.
- **Осторожно:** `кот~` может ловить лишнее (например, `который`). Для критичных слов используйте `{кот/коту/котом/коте/коту/коты/котов/котам~/котах}`.
- **Словари словоформ:** отдельный пакет с префиксом `word_` (например `word_cat`), содержащий все словоформы вручную. Используйте когда `~` недостаточно точен.
- **Префикс `word_`:** словари, содержащие полную таблицу словоформ (падежи, числа). Пример: `word_cat` = `кот`, `кота`, `коту`, `коте`, `котом`, `коты`, `кошек`...

## 16. Справочник DSL-конструкций

Полный список конструктивных тегов языка:

| Конструкция | Контекст | Назначение |
|---|---|---|
| `{вариант/--}` | правила | Inline-список с пустым вариантом |
| `[dict(name)]` / `[udict(name)]` | ответы | Вставка словаря (case-insensitive / case-sensitive) |
| `[&1]` + `[-{...}-]` | ответ + правило | Референция на выделенный фрагмент |
| `[%var=...]` | ответы | Присвоение/сброс переменной |
| `[if(!%var)]` / `[elsif(%var="...")]` | ответы | Условные блоки |
| `%last_mark` | ответы / условия | Мягкая контекстная метка |
| `--switch` / `--if` | ответы | Блок-условия (каждый тег — отдельный block) |
| `[el]` после `[ul]`/`[ol]` | ответы | Закрытие списка |
| `[inf(...)]{...}` | ответы | Генерация user-ссылок |
| `[@hvs("table", "row")]` | ответы | Вставка таблицы |
| `[action(name, param)]{...}` | ответы | Внешний вызов |
| `[@goto("Узел")]` / `[@extend("Узел")]` | ответы | Переход / переход с возвратом |
| `[disableautovars]` | ответы | Отключает автоматическое обновление переменных платформы при выдаче ответа. Используется в fallback-ответах для предотвращения потери контекста (`%that_anchor`, `%last_mark`) |
