# Беклог — Active Governed Backlog

**SSOT:** `docs/беклог.md` — единственный источник правды по незавершённым задачам, техдолгам и будущим улучшениям проекта. File is actively governed by the Orchestrator, updated automatically after every significant task via Post-Task Backlog Check.

**Last Audit:** 2026-05-28

## Lifecycle States

| Status | Описание |
|--------|----------|
| `OPEN` | Новая задача, не начата |
| `IN_PROGRESS` | Работа над задачей начата |
| `PARTIALLY_RESOLVED` | Задача выполнена частично — записан progress note |
| `BLOCKED` | Зависит от другой задачи / внешнего условия |
| `RESOLVED` | Задача выполнена — перенесён в Архив |
| `ARCHIVED` | Итоговое состояние закрытой задачи |

## Как работать с бэклогом

- Новые задачи добавлять в соответствующую категорию с уникальным ID (A-3, D-4, S-2, P-4 и т.д.) и статусом `OPEN`
- Inline TODO/FIXME/HACK в других файлах — **ЗАПРЕЩЕНО**. Ссылаться на ID задачи в этом файле: `# TODO: A-2`
- Оркестратор автоматически обновляет статусы, архивирует выполненные пункты, проверяет дубликаты — см. `agents/1_ORCHESTRATOR.md` §"Беклог — Active Governed Backlog"
- При частичном завершении — статус `PARTIALLY_RESOLVED`, в задаче записать residual work
