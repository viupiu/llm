# C. АРХИТЕКТУРА БОТА (BOT_ARCHITECTURE)

**SSOT для:** Структура бота, навыки, узлы, EVENT, CIAS, якоря, паттерн "Задаватель vs Обработчик", naming.

**НЕ содержит:** Синтаксис DSL (ссылка → A), Runtime-ограничения (ссылка → B), API (ссылка → F).

---

## 1. Типология узлов

| Тип | Назначение |
|-----|------------|
| **Контроллер** | Объединяет условие, логику (Skill) и ответ |
| **Контекстный** | `conditions` (`%that_anchor`) для обработки по-разному в зависимости от этапа |
| **Динамический** | Сложная логика в блоке ответов (`--if`, `--switch`) |
| **Композитный** | `[ExtendAnswer("...")]` для общих текстов |

---

## 2. Точки входа в сценарий

| Тип | Описание |
|-----|----------|
| **EVENT** | Системные триггеры в `dl_rules`: `EVENT <uuid>`. Поле JSON `events` обычно пустое. |
| **Приветственные** | Узлы, реагирующие на приветствия |
| **Триггерные** | Любой узел с правилами/примерами может стать точкой входа |
| **Fallback** | Узел с `*`. Обязателен для предотвращения молчания |

### Обязательные EVENT в голосовых ботах

| UUID | Назначение |
|------|------------|
| `00b2fcbe-f27f-437b-a0d5-91072d840ed3` | Загрузка (старт / приветствие) |
| `903f9438-5871-4160-80e0-32f418ca1169` | Тишина |
| `3f6b6289-4cf5-48da-b42e-6318899bbdd8` | Конец разговора |

EVENT-узлы в навыке `EVENTS` с условием `%that_anchor="CIAS"`.

## 3. ФАТАЛЬНО: Задаватель vs Обработчик

**Один узел НЕ может задать вопрос и отработать ответ!**

`[if($да)]` проверяет сущность в ТЕКУЩЕМ запросе. В узле-задавателе пользователь ещё НЕЧТО НЕ СКАЗАЛ.

**Правильный паттерн:**
- **Узел-задаватель:** задаёт вопрос, устанавливает якорь `[%that_anchor="имя"]`
- **Узлы-обработчики:** проверяют `%that_anchor="имя"` в `conditions` + сущность в правиле

Пример (эталон FPK):
```dl
Чем я могу Вам помочь? [%that_anchor="Чем я могу Вам помочь?"]

# В условиях обработчика:
conditions: %that_anchor="Чем я могу Вам помочь?"
```

## 4. Иерархия и структура (Навыки → Узлы)

### Навыки (Skill)
- **Регистрация:** JSON-файлы в `Skill` — только метаданные (ID, имя, статус).
- **Реализация:** Логика вынесена на бэкенд. БЗ хранит ссылку (UUID).
- **Классификация:**
  - Инфраструктурные: `start`, `events`
  - Бизнес-сценарии: по пунктам ТЗ
  - Диалоговые: `inf_dialog`
  - Песочница: `__TEST`

**Размер навыка:** 5–15 узлов. Клиент не обязан проходить навык линейно.

### Интеграция навыков (`Skill`)
- Поток: `DialogNode` → `skill_id` → [Бэкенд] → Результат → `DialogNode`

### Управление контентом (`Answer`)
- Полный декуплинг: тексты вынесены в отдельные JSON. `DialogNode` ссылается на UUID.
- Блочная структура: массив `blocks`.

## 4. Управление состоянием и контекстом

### Обычный якорь (`%that_anchor`)
Машина состояний: в ответе задаём этап, в `conditions` — тот же текст.

**Именование якорей (ОБЯЗАТЕЛЬНО):**

| Правило | Пояснение |
|---------|-----------|
| **Говорящий текст** | Значение якоря = формулировка вопроса этапа |
| **Совпадение с репликой** | Текст в якорях — побайтно одинаковый (регистр, пунктуация) |
| **Уникальность** | Если фраза встречается в разных контекстах — удлиняют уникальной частью |
| **Исключение** | Служебный супер-якорь **`CIAS`** — только EVENT / инфраструктура |
| **Запрещено** | `help_main`, `user_type_q`, `hotline_offer` и т.п. |

### Супер-якорь `CIAS`
Служебный супер-якорь для EVENT и глобальной инфраструктуры. Имеет приоритет над обычным якорем: если EVENT сработал при активном `%that_anchor="Укажите дату"`, узел с `CIAS` всё равно выиграет у fallback `*`.

```dl
conditions: %that_anchor="CIAS"
```

### `%last_mark`
Мягкий контекст (последняя тема). В `conditions`: `%last_mark="текст"`. В ответе: `[%last_mark="текст"]`.

### `[disableautovars]`
Пересохраняет якоря и метки на **текущем** шаге. Используются в тишине, fallback, повторе.

```dl
[disableautovars]
# Эквивалент:
[%that_anchor="[%that_anchor]"] [%last_mark="[%last_mark]"]
```

### `[nothing]`
Разрешает пустую реплику. Без него движок пропускает узел с пустым ответом.

```dl
[nothing]
[disableautovars]
```

### `voice.end_call`
Переменная сессии. `[%voice.end_call="True"]` — сигнал АТС.

## 5. Проектирование данных

### Стандары именования

| Сущность | Соглашение | Запрещено |
|----------|------------|-----------|
| **Словари** | `[префикс]_[категория]_[суть]` | — |
| **Переменные** | Короткие, семантические, **без префикса бота** | `%fpk_*`, избыточные префиксы |
| **Паки синонимов** | `keywords_<тема>` — однословные стеммы | Словарь как единственное правило |
| **Тип звонящего** | `user_type_physical`, `user_type_legal` | Весь матчинг в словаре без обёрток |
| **Согласие/отказ** | `common_yes`, `common_no`, `otkaz` (`reference/COMMON_DICTIONARIES.md`) | Новые `yes_dict` / `no_dict` |
| **Якоря** | Текст вопроса | `help_main`, `email_more`, `*_q` |

### Принцип: Правило узла vs именованный словарь

| Слой | Где | Что пишем |
|------|-----|-----------|
| **Скелет матчинга** | `dl_rules` узла | `*`, `[@cmb]`, `{термин~}`, `{{а/б}}`, `{[dict(пак)]}` |
| **Пак синонимов** | `## Словарь:` | Стеммы строками, без `=>`, `*`, `[@cmb]` |
| **Системные паки** | `common_yes`, `common_no`, `otkaz` | Готовые; в узле только `[dict(...)]` |

**Когда выносить в словарь:** синонимов > 3-5 в позиции или пак повторяется в нескольких узлах.
**Запрещено:** Переносить правила с `*`, `[@cmb]` в словарь.
**Допустимо:** правило вида `* [dict(name)] *` — скелет (`*`) присутствует.

## 6. Анти-паттерны: last_mark vs that_anchor

- **❌** `%that_anchor` для мягких уточнений («чай или кофе?»), когда достаточно `%last_mark`.
- **✅** `%last_mark` в `conditions` + `[%last_mark="..."]` в ответе.

## 7. Роль узла-перехода (`goto-only`)

- ❌ Считать узлы `[@goto]` "излишними".
- ✅ Узел-переход может быть архитектурной точкой: объединяет ветки делает схему понятнее, отражает вопрос в названии.

---

## 8. Атомарность узлов

- ✅ Одна реплика / одно намерение = один JSON-узел.
- ✅ DL-конструкции — только в `[]`, основной текст — БЕЗ скобок.
- ✅ Комментарии — отдельный block, `"inactive": true`, с `//`.
- ✅ Референции: `[&1]` только если в правиле есть `[-*-]` или `[-{a/b}-]`.
- ✅ Генерировать вариативность: точные фразы + шаблоны + словари.
- ✅ Атомарность: `[dict(name)]` вместо явного перечисления синонимов в правиле.

---

## 9. Глобальные перехватчики

Узлы с EVENT-условиями (`%that_anchor="CIAS"`) работают как глобальные перехватчики — они срабатывают вне зависимости от текущего диалогового этапа. Приоритет EVENT-узла с `CIAS` выше fallback-узла `*`.
