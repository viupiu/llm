# Чеклист голосового обзвонщика (эталоны)

Сводка по успешным ботам в `archives/examples/` (Demo Обзвоны Клининг, ФПК).  
 Подробности: `C_BOT_ARCHITECTURE.md`, `reference/COMMON_DICTIONARIES.md`, `A_LANGUAGE_SPEC.md`, `E_RESPONSE_FORMATTING.md` (SSML).

## Инфраструктура (обязательно)

- [ ] Навык **`EVENTS`**: узлы **Загрузка**, **Тишина**, **Конец разговора**
- [ ] В `dl_rules` тех же узлов — `EVENT` с UUID эталонов (см. `C_BOT_ARCHITECTURE.md`, раздел 2)
- [ ] У EVENT-узлов в `conditions`: **`%that_anchor="CIAS"`** (супер-якорь)
- [ ] Навык **`*`**: fallback с правилом `*`, ответ «не поняла» + `[disableautovars]` где нужно сохранить контекст
- [ ] Переменная **`voice.end_call`** и присвоение `[%voice.end_call="True"]` в «Конец разговора»
- [ ] Ветка **`master`** в `Branch/`
- [ ] `metadata.json` с `lc_version` (формат см. `D_GENERATION_RULES.md`)

## Сценарий

- [ ] Бизнес-навык(и) по смыслу ТЗ (отдельно исходящий/входящий — если в ТЗ два направления звонка)
- [ ] Этапы через **`%that_anchor`**: вопрос в ответе → дочерние узлы с тем же якорем в `conditions`
- [ ] Внутри этапа: узлы `Вопрос - вариант` и **`Вопрос - *`** для повтора при нераспознанном ответе
- [ ] Переходы **`[@goto("Имя узла")]`** / **`[@extend("...")]`** для общих блоков
- [ ] Словари из **`reference/COMMON_DICTIONARIES.md`** — переиспользовать, не изобретать `common_yes` / `otkaz` заново
- [ ] Для whitelist-полей (ФИО, дата из справочника): ветки **«есть в словаре»** / **«нет в словаре»** / **«повтор»** (паттерн ФПК)
- [ ] Узлы с пустым ответом по задумке — тег **`[nothing]`** (см. «Дублирование запроса» в эталоне ФПК)

## Голосовые приёмы

- [ ] **SSML** в прощании/конце звонка при необходимости паузы: `<speak>...<break time='1500ms'>...</speak>` + `[%voice.end_call="True"]` вне `speak` — см. `E_RESPONSE_FORMATTING.md`
- [ ] **`[disableautovars]`** на тишине, fallback, повторе — чтобы не сбросить якорь на этом шаге
- [ ] **`[%reply-]`** для узла «Повтори» (по необходимости)
- [ ] Сущности даты/времени: `$sys.time`, `dict(day/month/year)`, `switchdate`, `CalcTime` / `SetValue` — по ТЗ

## Не смешивать

- EVENT-триггер — в **`dl_rules`**, не в JSON-поле `events`
- **`CIAS`** — только для инфраструктуры и EVENT, не для шагов опроса
- **`=>` в словарях** — только если нужна нормализация в ответе (`[&dict(..., norm)]`)
