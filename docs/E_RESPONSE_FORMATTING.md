# E. ФОРМАТИРОВАНИЕ ОТВЕТОВ (RESPONSE_FORMATTING)

**SSOT для:** Форматирование ответов, теги выдачи, словари/udict/sudict в ответах, инлайн-условия, switch/case форматирование, модификаторы (:u:, :lu:), SSML, юзерлинки, MUST NOT форматирования.

**НЕ содержит:** Синтаксис DSL/функции/Action (→ A), Runtime семантику переходов/тегов (→ B), Архитектуру узлов (→ C), Правила генерации артефактов (→ D).

---

## 1. Паттерны ответов (`answers`)

### Форматирование

| Тег | Назначение |
|-----|------------|
| `[b]...[/b]` | Жирный |
| `[i]...[/i]` | Курсив |
| `[u]...[/u]` | Подчёркнутый |
| `[br]` | Перенос строки |
| `[ul]...[li]...[el]` | Ненумерованный список |
| `[ol]...[li]...[el]` | Нумерованный список |
| `[href(url)(_blank)]{текст}` | Ссылка |
| `[inf("скрытый")]{видимый}`| Юзерлинк |
| `[dict(name)]` | Случайная строка из словаря |
| `[udict(name)]`| Один раз за цикл |
| `[sudict(name)]`| Один раз за разговор |

### Переменные в ответах

Синтаксис `[%name]`, присвоения и сохранение сущностей см. `docs/A_LANGUAGE_SPEC.md` §23.

### Инлайн-условия

**Закрывающая `}` — в КОНЦЕ всего блока!**
```dl
[if($да)]{[%user_ready="true"] Хорошо. [@goto("Offer_Audit")]}
```

```dl
[if(!%user_name)]{
    Я не знаю, как тебя зовут
} [elsif(%user_name="Вася")]{
    Тебя зовут Васян
} [else]{
    [%user_name]
}
```

### Блочные условия

Только `--`, каждый тег — **отдельный** block в `answers.blocks`:

```json
{"text": "--switch([%var1])", "type": "unstyled"}
{"text": "--case(1)", "type": "unstyled"}
{"text": "Ответ для 1", "type": "unstyled"}
{"text": "--endswitch", "type": "unstyled"}
```

```dl
--switch([%var1])
--case(1)
Ответ для 1
--case(2)
Ответ для 2
--default
Дефолтный ответ
--endswitch
```

```dl
--if(!%var)
Ответ для --if(!%var) [%var="1"]
--elsif(%var="1")
Ответ для --elsif(%var="1")
--else
Ответ для --else [%var=""]
--endif
```

### Навигация: `[@goto]` и `[@extend]`

Подробное поведение переходов (`@goto`, `@extend`, `continue`, `break`) см. `docs/B_RUNTIME_MODEL.md` §9.

### Теги выдачи

Модификатор в **начале** строки (перед текстом и форматированием). Можно комбинировать.

| Модификатор | Поведение |
|---|---|
| `:u:`| Один раз за сессию |
| `:lu:`| Один раз за цикл; после прохода — сброс |
| `:last:`| Пониженный приоритет |
| `:+n:` / `:-n:`| Сдвиг веса |

**Синтаксис:** без скобок, с пробелом после двоеточия: `:lu: Текст`

### Глобальный ответ

```dl
[ExtendAnswer("Global_Name")]
```

### Пустой ответ

```dl
[%var="1"] [nothing]
```

### Задаватель vs Обработчик

Архитектурный паттерн см. `docs/C_BOT_ARCHITECTURE.md` §3.

### Служебные теги

Семантика `[disableautovars]`, `[nothing]`, `voice.end_call` см. `docs/B_RUNTIME_MODEL.md` §10.

### SSML в голосовом ответе

```dl
<speak>Рада была помочь. До свидания.<break time='1500ms'></speak> [%voice.end_call="True"]
```

### Функции

Синтаксис и счётчики см. `docs/A_LANGUAGE_SPEC.md` §24-25.

### Кликабельный текст (юзерлинк)

```dl
[inf("перейти к узлу X")]{Да}
```

### Заглушки

```dl
[Здесь будет [действие: перевод на оператора]]
```

## 3. Эталонные примеры ответов (из PATTERNS.md, архивов FPK)

### Переменные и присвоение
```
Отлично, доставка [%pizza_type] [%pizza_size]
Я запомню, что вы [&1] [%user_name="[&1]"]
Хорошо. [%user_name=""]
Извините, не расслышала. Повторите. [disableautovars]
До свидания. [%voice.end_call="True"]
```

### Словари в ответах
```
:lu: [b][dict(hello_ans)]![/b]
[udict(anec)]
```

### Списки и юзерлинки
```
Что любишь:
[ul] [inf("Чай")]{чай} [li] [inf]{кофе} [el] [%last_mark="чай или кофе?"]
```

### Референции и таблицы
```
Мы в [&dict(messenger, norm)]:
[br] [@hvs("messengers", "[&dict(messenger, norm)]")]
```

### Action
```
[action(weather, lon_lat="[&dict(1, norm)]")]{[%status="[[status]]"]}
```

### Навигация
```
[@goto("Узел 2.1 - Начало дерева")]
[ExtendAnswer("Глобальный ответ")]
```

## 4. MUST NOT (Форматирование и ответы)

1. ❌ HTML `<b>`, `<br>` — использовать `[b]`, `[br]`
2. ❌ backticks, `id="..."`
3. ❌ Все ответы в один inline-словарь
4. ❌ Закрывающая `}` не в конце блока if
5. ❌ `$` для переменных — только `%`
6. ❌ `== True`, `>=`, `++`, `--`. ❌ `@@`, `CRM_Trans:`. ❌ `$$prev`
7. ❌ `$var=value` ✅ `[%var="[$entity]"]`
8. ❌ Смешивание задавателя и обработчика (ссылка → `docs/C_BOT_ARCHITECTURE.md` §3)
9. ❌ Технические комментарии в DL-ответы
