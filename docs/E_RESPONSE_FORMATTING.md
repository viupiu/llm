# E. ФОРМАТИРОВАНИЕ ОТВЕТОВ (RESPONSE_FORMATTING)

**SSOT для:** Ответы бота, форматирование, теги выдачи, словари в ответах, переходы, паттерны из архива, задаватель vs обработчик.

**НЕ содержит:** Синтаксис правил (ссылка → A), Runtime (ссылка → B), Архитектуру (ссылка → C).

---

## 1. Паттерны ответов (`answers`)

### Форматирование

| Тег | Назначение |
|-----|------------|
| `[b]...[/b]` | Жирный |
| `[i]...[/i]` | Курсив |
| `[u]...[/u]` | Подчёркнутый |
| `[br]` | Перенос строки |
| `[ul]...[li]...[el]` | Ненумерованный список |
| `[ol]...[li]...[el]` | Нумерованный список |
| `[href(url)(_blank)]{текст}` | Ссылка |
| `[inf("скрытый")]{видимый}`| Юзерлинк |
| `[dict(name)]` | Случайная строка из словаря |
| `[udict(name)]`| Один раз за цикл |
| `[sudict(name)]`| Один раз за разговор |

### Переменные в ответах

| Конструкция | Описание |
|---|---|
| `[%name]` | Вывод |
| `[%name="значение"]`| Присвоение |
| `[%name=""]`| Сброс |
| `[%name="[$entity]"]`| Сохранение сущности ✅ |
| `[%name=$entity]`| Неправильно ❌ |

### Инлайн-условия

**Закрывающая `}` — в КОНЦЕ всего блока!**
```dl
[if($да)]{[%user_ready="true"] Хорошо. [@goto("Offer_Audit")]}
```

```dl
[if(!%user_name)]{
    Я не знаю, как тебя зовут
} [elsif(%user_name="Вася")]{
    Тебя зовут Васян
} [else]{
    [%user_name]
}
```

### Блочные условия

Только `--`, каждый тег — **отдельный** block в `answers.blocks`:

```json
{"text": "--switch([%var1])", "type": "unstyled"}
{"text": "--case(1)", "type": "unstyled"}
{"text": "Ответ для 1", "type": "unstyled"}
{"text": "--endswitch", "type": "unstyled"}
```

```dl
--switch([%var1])
--case(1)
Ответ для 1
--case(2)
Ответ для 2
--default
Дефолтный ответ
--endswitch
```

```dl
--if(!%var)
Ответ для --if(!%var) [%var="1"]
--elsif(%var="1")
Ответ для --elsif(%var="1")
--else
Ответ для --else [%var=""]
--endif
```

### Словари в ответах

| Тег | Поведение |
|-----|-----------|
| `[dict(name)]`| Разовый; синонимы; ответы — каждый на отдельной строке; без `id="..."`, без backticks. |
| `[udict(name)]`| по разу за цикл, счётчик сбрасывается |
| `[sudict(name)]`| один раз за разговор |

### Навигация: `[@goto]` и `[@extend]`

| Сценарий | Исходный ответ | Целевой ответ | Итог | Пользователь в |
|---|---|---|---|---|
| goto | `Привет! [@goto("2")] Как дела?` | `Мой друг!` | `Привет! Мой друг!` | целевом |
| goto + continue | то же | `Мой друг! [continue]` | `Привет! Мой друг! Как дела?` | целевом |
| goto + break | то же | `Мой друг! [break]` | `Привет! Как дела?` | исходном |
| extend | `Привет! [@extend("2")] Как дела?` | `Мой друг!` | `Привет! Мой друг! Как дела?` | исходном |
| extend + break | то же | `Мой друг! [break]` | `Привет! Как дела?` | исходном |

### Теги выдачи

Модификатор в **начале** строки (перед текстом и форматированием). Можно комбинировать.

| Модификатор | Поведение |
|---|---|
| `:u:`| Один раз за сессию |
| `:lu:`| Один раз за цикл; после прохода — сброс |
| `:last:`| Пониженный приоритет |
| `:+n:` / `:-n:`| Сдвиг веса |

**Синтаксис:** без скобок, с пробелом после двоеточия: `:lu: Текст`

### Глобальный ответ

```dl
[ExtendAnswer("Global_Name")]
```

### Пустой ответ

```dl
[%var="1"] [nothing]
```
```dl
[nothing]
[disableautovars]
```

### Задаватель vs Обработчик

**Один узел НЕ может задать вопрос и отработать ответ!**

- **Задаватель** (`Intro_Company`): вопрос + `[%that_anchor="имя"]`
- **Обработчик** (`Intro_Yes`): `%that_anchor="имя"` в `conditions` + сущность в правиле

### Служебные теги

| Тег | Когда |
|---|---|
| `[disableautovars]` | Продлить жизнь якоря/метки |
| `[nothing]` | Намеренно пустая реплика |
| `[%voice.end_call="True"]` | Завершение звонка |

### SSML в голосовом ответе

```dl
<speak>Рада была помочь. До свидания.<break time='1500ms'></speak> [%voice.end_call="True"]
```

### Функции (не изобретать!)

```dl
[@Date()]
[@Time()]
[@Concatenate("A", "B")]
[@Greater("varname", "2")]
[@Equal("varname", "value")]
[@Set("varname", "...", "HIDE")]
```

❌ `@Add`, `[%var@]`, `@"1"`, `[%var++]`

Счётчики увеличиваются присваиванием:
```dl
[if(!%cnt)] Не понял. [%cnt="1"]
[if(%cnt="1")] Не понял. [%cnt="2"]
```

### Кликабельный текст (юзерлинк)

```dl
[inf("перейти к узлу X")]{Да}
```

### Заглушки

```dl
[Здесь будет [действие: перевод на оператора]]
```

## 2. ФОРМАТ УЗЛОВ (Ответы) — только для DL Автора Ответов

```
## Узел: ИмяУзла

Условия:
%that_anchor="имя_якоря"

Ответы:
Текст первого варианта ответа.
Текст второго варианта ответа.
```

- **НЕ ПИСАТЬ** блок `Правила:` — зона DL Автора Правил.
- Обязательно пишите блок "Условия:" (если нет — "Нет").
- Каждый ответ — отдельная строка.
- Inline-словарь `{Да / Хорошо}` — только для коротких синонимов.

### Тег `:lu:`

```
:lu: Ответ 1.
:lu: Ответ 2.
```
- Без скобок: `:lu:`
- С пробелом: `:lu: Текст`
- Можно с условиями: `:lu: [if(%user_name)]{Привет, [%user_name]}`

## 3. Эталонные примеры (из PATTERNS.md, архивов FPK)

### Переменные и присвоение
```
Отлично, доставка [%pizza_type] [%pizza_size]
Я запомню, что вы [&1] [%user_name="[&1]"]
Хорошо. [%user_name=""]
Извините, не расслышала. Повторите. [disableautovars]
До свидания. [%voice.end_call="True"]
```

### Словари в ответах
```
:lu: [b][dict(hello_ans)]![/b]
[udict(anec)]
```

### Инлайн-условия
```
[if(!%user_name)]{
    Я не знаю, как тебя зовут
} [elsif(%user_name="Вася")]{
    Тебя зовут Васян
} [else]{
    [%user_name]
}
```

### Блочные условия
```
--switch([%var1])
--case(1)
Ответ 1
--default
Дефолт
--endswitch
```

### Списки и юзерлинки
```
Что любишь:
[ul] [inf("Чай")]{чай} [li] [inf]{кофе} [el] [%last_mark="чай или кофе?"]
```

### Референции и таблицы
```
Мы в [&dict(messenger, norm)]:
[br] [@hvs("messengers", "[&dict(messenger, norm)]")]
```

### Action
```
[action(weather, lon_lat="[&dict(1, norm)]")]{[%status="[[status]]"]}
```

### Навигация
```
[@goto("Узел 2.1 - Начало дерева")]
[ExtendAnswer("Глобальный ответ")]
```

## 4. MUST NOT (Форматирование и ответы)

1. ❌ HTML `<b>`, `<br>` — использовать `[b]`, `[br]`
2. ❌ backticks, `id="..."`
3. ❌ Все ответы в один inline-словарь
4. ❌ Закрывающая `}` не в конце блока if
5. ❌ `$` для переменных — только `%`
6. ❌ `== True`, `>=`, `++`, `--`. ❌ `@@`, `CRM_Trans:`. ❌ `$$prev`
7. ❌ `$var=value` ✅ `[%var="[$entity]"]`
8. ❌ Смешивание задавателя и обработчика
9. ❌ Технические комментарии в DL-ответы
10. ❌ `backticks` / `id`
