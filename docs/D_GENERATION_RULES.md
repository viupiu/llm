# D. ПРАВИЛА ГЕНЕРАЦИИ (GENERATION_RULES)

**SSOT для:** Где и как писать DSL в выходных артефактах, форматирование `dl_rules_result.md`, анти-паттерны генерации.

**НЕ содержит:** Синтаксис DSL (→ `docs/A_LANGUAGE_SPEC.md`), Runtime (→ `docs/B_RUNTIME_MODEL.md`), Архитектуру (→ `docs/C_BOT_ARCHITECTURE.md`).

---

## 1. Карта размещения DSL

Где разрешено использовать DSL-конструкции, а где — строго JSON:

| Поле | DSL разрешён | Запрещено |
|------|-------------|-----------|
| `DialogNode.dl_rules.blocks[].text` | `*`, `~`, `{/}`, `[@cmb]`, `[dict]`, `[- -]` | Регулярки, HTML |
| `DialogNode.answers.blocks[].text` | `[%var]`, `[if]{}`, `--if`, `[@goto]`, `[dict]`, `[b]`, `[br]` | `<b>`, `<br>`, raw HTML |
| `DialogNode.conditions` | `%that_anchor`, `%last_mark` | DSL-паттерны (тут — JSON-строки) |
| `Dictionary.content.blocks[].text` | Канон, `=>синоним~` (в `_norm`) | `*`, `[@cmb]`, `=>` в обычном паке |
| Слоты / UUID / связи | — | DSL-конструкции |

---

## 2. Формат выходного файла `dl_rules_result.md`

- ✅ Файл — только `work/<BotSlug>/dl_rules_result.md`. Запрет `rules.md`, `intent_rules.md`.
- ✅ Только секции `## Узел: <Name>` и `## Словарь: <Name>`. Запрет прозы между секциями.
- ✅ В узле — **скелет** (`* [@cmb({термин~}, {[dict(пак)]})] *`), а не единственный `[dict(key)]`.
- ✅ В `## Словарь:` — только стеммы, по одной на строку. Без `*`, `[@cmb]`, `=>` (кроме `_norm`).
- ✅ Явный заголовок формата: `## Словарь: <имя_пака>`.

---

## 3. Блоковые условия: анти-паттерны

- ❌ `–switch`, `–if` (длинное тире/ен-даш). ✅ `--switch`, `--if` (два дефиса).
- ❌ Весь switch в одной строке. ✅ Каждый `--case`, `--endswitch`, `--if`, `--endif` — отдельный block.

---

## 4. Ответы: анти-паттерны

- ❌ HTML `<b>`, `<br>` в ответах. ✅ `[b]`, `[br]`, `[ul]`/`[li]`/`[el]`.
- ❌ `[ul]` без `[el]`. ✅ `[ul]...[li]...[el]`.
- ❌ Сбор слотов через `[if]`. ✅ Слоты — только JSON (`slot_filling`).
- ❌ Вывод слова "default". ✅ Текст в `[default]` — полноценный ответ.
- ❌ Путать `%var` и `$entity`. ✅ `[if($city)]` — сущность, `[if(%user_name)]` — переменная.

---

## 5. Словари: анти-паттерны

- ❌ Сразу `=>синоним~` без канона. ✅ Канон — первая строка без `=>`.
- ❌ Тильды/звёздочки в каноне (`IT~`, `*Москва*`). ✅ Канон — чистый текст.
- ❌ `=>` в ненормализующем паке. ✅ `=>` только в `*_norm`.
- ❌ `*`, `[@cmb]` внутри `## Словарь:`. ✅ Только стеммы, по одной на строку.

---

## 6. Референции: анти-паттерны

- ❌ `[&1]` без `[- -]` в правиле. ✅ `[- -]` / `[-{a/b}-]` в правиле → `[&1]` в ответе.

---

## 7. Паттерны поиска: анти-паттерны

- ❌ Regex в `dl_rules`. ✅ `[@cmb({[dict(name)]}, {вариант})]`.
- ❌ DSL для слотов в `dl_rules`. ✅ Слоты — только JSON.

---

## 8. Директивы перехода: анти-паттерны

- ❌ `[@extend]` вместо `[@goto]` для постоянного перевода. ✅ `[@goto]` — остаёмся, `[@extend]` — возвращаемся.
- ❌ Ждать текста после `[@goto]` без `[continue]` в целевом узле. ✅ Использовать `[@extend]` или добавить `[continue]`.
- ❌ Путать `[ExtendAnswer("x")]` и `[@extend("Узел")]`. ✅ ExtendAnswer — глобальный шаблон.
- ❌ Директивы без скобок и кавычек. ✅ `[@goto("Имя узла")]`.

## 10. Узлы: правила генерации

→ См. `docs/C_BOT_ARCHITECTURE.md` §8 (атомарность узлов).

## 12. Именование: анти-паттерны

- ❌ Узлы/навыки `Узел 1.1`, `Навык 1` — недопустимы. ✅ Смысловое имя: `Приветствие`, `Заказ пиццы`.

## 13. Slot Filling: анти-паттерны (авторинг)

- ❌ `slot_filling` с `required: true` без `question` или `variable`. ✅ Всегда задавать вопрос и переменную.
- ❌ Слоты без `retry_count` и `failure_answer`. ✅ Указывать лимит попыток и ответ на крайний случай.

## 14. Словари: дополнительные правила

- ❌ `[dict]` выдача без нормализации, но с `=>` в словаре. ✅ Если словарь нормализующий, использовать `[&dict(name, norm)]`.
- ❌ `[dict]` в текстовом поле без референции на файл словаря. ✅ Каждое `[dict(name)]` → существует файл `Dictionary/{id}.json` с соответствующим именем.

---

## 11. ФОРМАТ УЗЛОВ (Ответы) — для DL Автора Ответов

```
## Узел: ИмяУзла

Условия:
%that_anchor="имя_якоря"

Ответы:
Текст первого варианта ответа.
Текст второго варианта ответа.
```

- **НЕ ПИСАТЬ** блок `Правила:` — зона DL Автора Правил.
- Обязательно пишите блок "Условия:" (если нет — "Нет").
- Каждый ответ — отдельная строка.
- Inline-словарь `{Да / Хорошо}` — только для коротких синонимов.
